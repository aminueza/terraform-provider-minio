# minio_config_restore (Resource)

Restores MinIO configuration from a previous point in history. Use with minio_config_history data source to identify restore points.

## Example Usage

```terraform
data "minio_config_history" "recent" {
  limit = 10
}

resource "minio_config_restore" "rollback" {
  restore_id = data.minio_config_history.recent.entries[0].restore_id
}
```

## Import

Import is supported using the restore ID:

```shell
terraform import minio_config_restore.example restore-id
```

## Notes

- This resource uses `RestoreConfigHistoryKV` from the madmin-go SDK
- Use the `minio_config_history` data source to list available restore points
- Restoring a config is a one-time action; the resource just records which restore ID was used
- Deleting the resource from Terraform state does not affect the actual MinIO config
- A server restart may be required after config restoration for changes to take effect
