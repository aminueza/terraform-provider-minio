# minio_s3_bucket_replication_resync (Resource)

Triggers a resync of existing objects for a bucket replication rule. This replays all objects that existed before replication was configured or that failed to replicate.

This resource triggers a one-time resync operation. Creating it kicks off the resync; destroying it is a no-op. To re-trigger a resync, taint or replace the resource.

## Example Usage

```terraform
resource "minio_s3_bucket_replication_resync" "sync_existing" {
  bucket = minio_s3_bucket.source.id

  depends_on = [minio_s3_bucket_replication.to_target]
}
```
